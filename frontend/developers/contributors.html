How to Contribute
Participation
guidelines.
1
Start from the repository
Clone or fork the RoomMate repository before starting any contribution work.
2
Use a separate branch
Keep your work organized by creating a branch for the feature, fix, or update you are working on.
3
Keep the design consistent
When editing website pages, use the same fonts, colors, spacing, card style, and overall layout used across the main site.
4
Submit a clear pull request
After making changes, commit them with a meaningful message and open a pull request describing your work.

Code of
conduct.

Our Commitment

RoomMate is committed to providing a respectful, welcoming, and collaborative environment for everyone who participates in the project.

Contributors, users, and reviewers should feel comfortable asking questions, sharing ideas, reporting issues, and discussing improvements.

All participants are expected to communicate respectfully, give constructive feedback, respect different opinions and experience levels, and keep discussions focused on improving the project.

Harassment, insults, discrimination, offensive language, hostile communication, repeated disruption, and sharing private information without permission are not allowed.

Project Reuse
Starting from what we had,
then improving it.

RoomMate was reused from a previous academic project and then transformed into a more complete open-source project with documentation, a public website, contributor guidance, and project standards.

Starting from What We Had

The project reused as a starting point was the earlier ITSE305-Project. Instead of building everything from scratch, the existing work was taken and improved.

The new open-source version, ITSE476_project, adds public project pages, contribution guidelines, a code of conduct, and clearer organization for community access.

This directly follows the requirement of starting from an existing project and showing how it was transformed into an open-source system.
