// Reserve a room for a user
public String reserveRoom(
    User user, Room room) {

  if (!room.isAvailable()) {
    return "Room " + room.getNumber()
         + " is already reserved.";
  }

  room.setAvailable(false);
  reservations.add(
    "User " + user.getName()
    + " reserved room "
    + room.getNumber()
  );

  return "Room " + room.getNumber()
    + " reserved for "
    + user.getName() + ".";
}

What It Does
Everything you need.
Nothing you don't.

RoomMate is intentionally lean. A clean Java codebase that anyone can read, understand, and extend.

Room Search
Search available rooms by type, price range, or availability. Input validation and clear error messages built in.
Room Reservation
Reserve rooms with user details. Handles conflicts, validates emails, and confirms bookings immediately.
User Management
Create and manage users with name and email validation. Auto-incrementing IDs with full getter/setter support.
Automated Testing
Full JUnit 5 test suite covering constructors, setters, getters, and all search and reservation logic.
CI/CD Pipeline
GitHub Actions compiles and runs all tests automatically on every pull request to main.
Open Source
MIT licensed. Fork it, extend it, use it freely. All contributions are welcome without restriction.

About
Why RoomMate?

Built for Learning. Built to Share.

RoomMate started as an academic project at the University of Bahrain (ITSE305) and was transformed into a fully open-source project as part of ITSE476 — Free and Open-Source Software Development.

The goal is to provide a clean, well-tested, and well-documented Java codebase that students and developers can freely learn from, contribute to, and build upon.

The name reflects both its purpose and spirit: a system for managing room bookings, built by a team — your roommate in hotel management software.

Starting from What We Had

RoomMate didn't start from scratch. The core system — Room.java, User.java, ReserveRoom.java — was originally built for ITSE305 (Software Engineering) at the University of Bahrain. It included JUnit 5 tests and a GitHub Actions CI/CD pipeline.

For ITSE476 (Free and Open-Source Software Development), we took that foundation and transformed it into a proper open-source project: adding a license, documentation, contribution guidelines, a public website, and opening it to the community.
