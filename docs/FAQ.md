# Frequently Asked Questions (FAQ)
This document provides answers to common questions about RoomMate — an open-source hotel reservation system.

---

## General Questions

### What is RoomMate?
RoomMate is an open-source hotel reservation system that allows users to search for available rooms, make bookings, and manage their reservations efficiently.

### Who is this project for?
RoomMate is designed for educational purposes and small-scale hotel booking scenarios. It is especially suitable for students learning software development and open-source practices.

### Is RoomMate free to use?
Yes, RoomMate is released under the MIT License, which allows free use, modification, and distribution.

---

## Using the System

### How can I use the system?
Users can register an account, log in, browse available rooms, and complete reservations through the system interface.

### Do I need an account to make a reservation?
Yes, users are required to create an account before making reservations.

### Can I cancel a reservation?
Yes, reservations can be managed and canceled through the user dashboard (depending on the implemented features in each phase).

### Is there a graphical user interface (GUI)?
Currently, the system is console-based. A graphical interface is planned for future development phases.

---

## Project Status

### Is the project complete?
No, the project is currently in Phase 1. Additional features and improvements will be added in future phases.

### What is being developed in this phase?
The focus is on setting up the open-source structure, documentation, and basic frontend pages.

## Reporting Issues

### How can I report a bug or issue?
You can report issues using [GitHub Issues](https://github.com/FatimaxYusuf/ITSE476_project/issues).

### What information should I include in a bug report?
Please include:
- A clear description of the issue  
- Steps to reproduce the problem  
- Expected vs actual behavior
